# Architecture Decision Records

Use ADRs to record durable technical and workflow decisions that future tasks
need to understand.

ADRs explain why the project chose an approach. Operational steps belong in
agent guides, API details belong in `docs/api-contract.md`, and setup details
belong in `README.md`.
